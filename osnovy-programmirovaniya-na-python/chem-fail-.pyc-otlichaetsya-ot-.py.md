# Чем файл .pyc отличается от .py?

Файлы **`.py`** и **`.pyc`** — это разные стадии одного и того же Python-кода, но с разным назначением.

***

### **1. `.py` — исходный код**

* Это **текстовый файл**, который мы пишем на Python.
* Содержит инструкции в виде читаемого человеком кода.
*   Пример:

    ```python
    print("Hello, world!")
    ```

***

### **2. `.pyc` — байт-код Python**

* Это **скомпилированная версия** `.py` файла.
* Создаётся автоматически при первом импорте модуля (или при запуске, если кэширование включено).
*   Хранится в папке `__pycache__/` и обычно называется с указанием версии Python:

    ```
    __pycache__/module.cpython-311.pyc
    ```
* Не содержит исходного кода, а хранит **байт-код** — инструкции, которые исполняет виртуальная машина Python (CPython).

***

### **3. Зачем нужен `.pyc`**

* Ускоряет запуск программы, потому что Python не нужно заново парсить `.py` в байт-код.
* При следующем импорте модуля Python загружает готовый `.pyc`, если:
  * исходный `.py` не изменился,
  * версия Python совпадает.

***

### **4. Отличия `.py` и `.pyc`**

| `.py` (исходник)            | `.pyc` (байт-код)                    |
| --------------------------- | ------------------------------------ |
| Читаем человеком            | Читаем только Python-интерпретатором |
| Текст                       | Двоичный формат                      |
| Мы пишем вручную            | Создаётся автоматически              |
| Не зависит от версии Python | Зависит от версии Python             |
| Можно редактировать         | Нельзя редактировать напрямую        |

***

### **5. Важно для собеседования**

* `.pyc` — это **не** машинный код, а промежуточный байт-код для CPython.
* Даже без `.pyc` программа запустится — Python просто заново скомпилирует `.py`.
* Можно удалить папку `__pycache__` — это безопасно.
* `.pyc` ускоряет только **импорт модулей**, а не саму логику выполнения.
