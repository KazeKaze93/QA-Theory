# Есть приложение типа мессенджера, пользователь заходит в чат и отсылает файл ...

**Баг:**

1. **Неудачная отправка с приложением в рабочем состоянии:**
   * Если пользователь видит сообщение "Failed to send..." при каждой попытке отправить файл, и приложение находится в нормальном рабочем состоянии, то это может быть признаком бага. В этом случае, неудачная отправка должна быть расследована и устранена.
2. **Проблемы с интерфейсом:**
   * Если пользователь следует всем указаниям для отправки файла, но все равно видит сообщение об ошибке, возможно, есть проблемы с интерфейсом или обратной связью, что также может быть считано как баг.
3. **Отсутствие информации о причине:**
   * Если сообщение об ошибке не предоставляет достаточной информации о причине неудачной отправки (например, не указывает на проблемы с соединением, форматом файла и т. д.), это может считаться багом.

**Не баг:**

1. **Отправка в нерабочих условиях:**
   * Если пользователь пытается отправить файл в условиях, когда отправка файлов недоступна (например, из-за проблем с сетью), сообщение "Failed to send..." может быть предупреждением, а не багом.
2. **Файл слишком большой:**
   * Если приложение имеет ограничение на размер файлов для отправки, и пользователь пытается отправить файл, который превышает этот лимит, сообщение об ошибке также может быть оправданным и не считаться багом.
3. **Проблемы на стороне сервера:**
   * Если сервер, к которому отправляется файл, недоступен или не может обработать запрос, сообщение об ошибке также может быть следствием проблем на стороне сервера, а не багом в самом приложении.
