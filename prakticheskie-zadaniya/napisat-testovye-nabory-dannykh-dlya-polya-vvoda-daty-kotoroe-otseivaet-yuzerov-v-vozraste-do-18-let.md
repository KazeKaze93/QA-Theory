# Написать тестовые наборы данных для поля ввода даты, которое отсеивает юзеров в возрасте до 18 лет

Вот подробный чек-лист для тестирования поля ввода даты, которое отсеивает пользователей младше 18 лет:

1. **Корректные данные:**
   * [ ] Ввести дату рождения пользователя, который старше 18 лет.
   * [ ] Проверить, что система принимает корректную дату.
2. **Точно 18 лет:**
   * [ ] Ввести дату рождения пользователя, который точно 18 лет.
   * [ ] Проверить, что система принимает эту дату.
3. **Младше 18 лет:**
   * [ ] Ввести дату рождения пользователя, который младше 18 лет.
   * [ ] Проверить, что система отсеивает пользователя и выводит соответствующее сообщение.
4. **Некорректный формат даты:**
   * [ ] Ввести дату в некорректном формате.
   * [ ] Проверить, что система обрабатывает некорректный формат корректно.
5. **Пустое поле:**
   * [ ] Оставить поле ввода даты пустым.
   * [ ] Проверить, что система обрабатывает пустое поле корректно.
6. **Отрицательный сценарий (если применимо):**
   * [ ] Ввести недопустимую дату, которая не соответствует возрасту пользователя.
   * [ ] Проверить, что система отсеивает пользователя и выводит соответствующее сообщение.
7. **Граничные случаи:**
   * [ ] Ввести дату, соответствующую последнему дню 17 лет.
   * [ ] Проверить, что система отсеивает пользователя и выводит соответствующее сообщение.
   * [ ] Ввести дату, соответствующую первому дню 18 лет.
   * [ ] Проверить, что система принимает эту дату.
8. **Случай с будущей датой:**
   * [ ] Ввести текущую или будущую дату.
   * [ ] Проверить, что система корректно обрабатывает будущие даты.
9. **Случай с некорректным годом:**
   * [ ] Ввести дату с некорректным годом (например, 01.01.5000).
   * [ ] Проверить, что система обрабатывает некорректный год корректно.
