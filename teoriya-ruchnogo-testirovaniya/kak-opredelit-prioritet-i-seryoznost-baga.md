# Как определить приоритет и серьёзность бага

**Приоритет (Priority)** — это **насколько срочно баг нужно исправить**.\
**Серьёзность (Severity)** — это **насколько баг критичен для работы системы**.

| Параметр        | Описание                    | Пример                                                                  |
| --------------- | --------------------------- | ----------------------------------------------------------------------- |
| **Серьёзность** | Влияние на функциональность | Критичный баг: невозможность оплатить заказ → высокий severity          |
| **Приоритет**   | Срочность исправления       | Срочно: баг на странице оплаты → высокий priority, даже если баг редкий |

#### **Основные уровни:**

* **Severity:** Blocker, Critical, Major, Minor, Trivial
* **Priority:** High, Medium, Low

#### **Примеры комбинаций:**

* **High Severity + High Priority** → система падает, надо исправлять немедленно.
* **Low Severity + Low Priority** → косметическая ошибка, исправление можно отложить.
* **High Severity + Low Priority** → критичный функционал, но баг возникает редко, можно исправить позже.
* **Low Severity + High Priority** → косметическая ошибка на главной странице → исправление быстро.
