# Какая разница между Scrum и Kanban?

### **1. Основной подход**

| Аспект      | Scrum                                                | Kanban                                       |
| ----------- | ---------------------------------------------------- | -------------------------------------------- |
| **Процесс** | Итеративный, с фиксированными спринтами (1–4 недели) | Непрерывный поток задач, без спринтов        |
| **Цель**    | Доставить готовый инкремент продукта в конце спринта | Постоянная доставка задач по мере готовности |

***

### **2. Планирование и ритуалы**

| Аспект       | Scrum                                                      | Kanban                                                                 |
| ------------ | ---------------------------------------------------------- | ---------------------------------------------------------------------- |
| Планирование | Спринт-планирование, Daily Stand-up, Review, Retrospective | Нет обязательных ритуалов, часто используют только ежедневные стендапы |
| Итерации     | Фиксированная длина спринта                                | Нет фиксированных итераций, работа ведется непрерывно                  |

***

### **3. Роли**

| Аспект          | Scrum                                           | Kanban                                                     |
| --------------- | ----------------------------------------------- | ---------------------------------------------------------- |
| Роли            | Product Owner, Scrum Master, команда разработки | Нет обязательных ролей, структура гибкая, может быть любой |
| Ответственность | Чётко распределена между ролями                 | Ответственность распределяется по команде, гибко           |

***

### **4. Управление задачами**

| Аспект                        | Scrum                                         | Kanban                                                                |
| ----------------------------- | --------------------------------------------- | --------------------------------------------------------------------- |
| Планирование работы           | На основе выбранного спринта (Sprint Backlog) | Работа вытягивается по мере готовности, лимиты WIP (Work In Progress) |
| Изменения во время выполнения | Обычно запрещены в спринте                    | Можно менять задачи в любой момент                                    |
| Визуализация                  | Scrum Board для спринта                       | Kanban Board с колонками "To Do – In Progress – Done" и лимитами WIP  |

***

### **5. Метрики**

| Аспект           | Scrum                    | Kanban                            |
| ---------------- | ------------------------ | --------------------------------- |
| Основные метрики | Velocity, Burndown chart | Lead time, Cycle time, Throughput |
