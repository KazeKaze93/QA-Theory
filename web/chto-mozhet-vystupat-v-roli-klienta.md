# Что может выступать в роли клиента?

В роли клиента в **клиент-серверной архитектуре** может выступать любое приложение или устройство, которое отправляет запрос к серверу для получения или изменения данных.

***

### Основные примеры клиентов:

1. **Веб-браузеры**
   * Chrome, Firefox, Safari, Edge — при вводе URL или работе с веб-приложениями.
2. **Мобильные приложения**
   * iOS/Android приложения (например, мобильный банк, Telegram, YouTube).
3. **Desktop-приложения**
   * Outlook (почта), Steam (игровой клиент), Slack (рабочий мессенджер).
4. **Автоматизированные системы и скрипты**
   * Тестовые фреймворки (Selenium, Playwright, Postman).
   * Python-скрипты с `requests`, `aiohttp`.
5. **IoT-устройства (умные устройства)**
   * Умные колонки, датчики, камеры — запрашивают данные у серверов.
6. **Другое ПО/сервисы**
   * Один микросервис может быть клиентом по отношению к другому.
   * Например, сервис заказов обращается к сервису пользователей.
