# Как работает DNS?

**DNS (Domain Name System)** — это распределённая система, которая переводит понятные человеку доменные имена (например, `example.com`) в IP-адреса серверов (например, `93.184.216.34`), чтобы браузер или приложение могли установить соединение.

***

### Основные шаги работы DNS-запроса

* Ты вводишь адрес сайта в браузере.
* Компьютер сначала смотрит: нет ли IP в кэше (в памяти).
* Если нет — спрашивает у DNS-сервера (обычно у провайдера).
* DNS-сервер по цепочке ищет нужный адрес (корневые → TLD → авторитетный сервер).
* Возвращает IP-адрес.
* Браузер соединяется с сервером по IP.

***

### Виды DNS-записей

* **A** — IPv4-адрес.
* **AAAA** — IPv6-адрес.
* **CNAME** — алиас (псевдоним домена).
* **MX** — почтовый сервер.
* **TXT** — текстовые данные (например, SPF, DKIM).
* **NS** — указывает на DNS-серверы домена.

***

### Для QA важно

* Проверка правильности настроек DNS-записей (A, CNAME, MX).
* Проверка времени отклика DNS.
* Тестирование кэширования и обновления записей (TTL).
* Проверка работы домена по HTTP и HTTPS (в т.ч. `www` и без).
