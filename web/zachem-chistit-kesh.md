# Зачем чистить кэш?

Чистка кэша нужна для того, чтобы браузер или система использовали **актуальные данные**, а не старые версии ресурсов.

***

### 1️⃣ Основные причины чистки кэша

1. **Обновления сайта или приложения**

* Если CSS, JS или изображения были изменены на сервере, браузер может показывать старую версию из кэша.
* Пример: кнопка изменила цвет, но пользователь видит старый стиль.

2. **Ошибки отображения**

* Иногда из-за конфликтов между старым кэшем и новым кодом страница может «ломаться» — некорректное отображение, нерабочие скрипты.

3. **Тестирование веб-приложений**

* QA нужно проверить, как сайт работает с **чистого состояния**, чтобы убедиться, что новые изменения применяются корректно.
* Проверка авторизации, сессий, работы API без влияния старых кэшированных данных.

4. **Проблемы с данными и cookies**

* Иногда старые кэшированные данные мешают корректной работе функционала, например, корзины или форм.

***

### 2️⃣ Виды кэша, который очищают

* **Браузерный кэш:** HTML, CSS, JS, изображения.
* **Application cache / Service Worker:** данные веб-приложений.
* **Cookies и локальное хранилище:** данные сессий и настроек.

***

### 3️⃣ Практический эффект

* После очистки кэша браузер заново загружает все файлы с сервера.
* Это позволяет убедиться, что приложение работает **с актуальными данными**, а не с устаревшими.
